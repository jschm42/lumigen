# Lumigen v0.3.0-beta

Third beta release of **Lumigen**, focusing on expanded provider support, advanced data management, and workflow optimizations.

> [!NOTE]
> Beta release: major features including multi-provider support and data portability are now integrated. While stabilized, minor refinements to the internal data schema may occur before the stable release.

## Highlights

- **Expanded Provider Ecosystem**: Native integration for **Google** and **FAL.ai** image generation providers, including specialized resolution and aspect ratio handling.
- **Advanced Data Portability**: Comprehensive JSON Import/Export system for Admin Profiles, Models, and Styles to facilitate configuration sharing and backups.
- **Dynamic UX**: Responsive generation frames that reflect selected aspect ratios in real-time and an improved modular gallery with persistent sizing.
- **Session Archiving**: One-click "Download Session" feature to export a complete ZIP archive containing all images and a structured Markdown summary of the session.
- **Admin Dashboard Refinement**: Modernized admin interface using HTMX for seamless profile and model management without full page reloads.

## What's New

### Image Generation & Providers

- **Google Integration**: Full support for Google's image generation models with custom aspect ratio and resolution settings.
- **FAL.ai Support**: Integration of FAL.ai as both an image generation and upscaling provider.
- **Robust Job Orchestration**: Improved service layer for managing multi-provider generation jobs with enhanced error handling, status tracking, and timeout management.
- **Input Locking**: Generation form inputs are automatically locked during active generation to prevent accidental duplicate submissions.

### Data Management & Admin

- **Import/Export System**: Added JSON-based import and export capabilities for administrative entities (Profiles, Models, Styles).
- **Style Management**: New Style definition system with CRUD support, allowing styles to be defined with original prompts and metadata.
- **Prompt Enhancement**: Configurable default enhancement prompts at the provider level.
- **Version Tracking**: Integrated version management system that displays the current release version in the application header.

### User Interface & Experience

- **Aspect Ratio Preview**: The generation frame now dynamically adjusts its proportions based on the selected aspect ratio before generation begins.
- **Quick Iteration**: Added a "Re-Generate" button directly on user prompt bubbles in the chat history.
- **Gallery Optimizations**: 
  - Persistent thumbnail size preferences (Small, Medium, Large).
  - Modular grid fragments for faster loading and improved code maintainability.
- **Mobile Enhancements**: Optimized compact layout for the workspace sidebar on mobile devices.
- **Accessibility & Polish**: Sticky headers for the generation form, improved focus states, and overall UI refinement across both light and dark modes.

### Quality & Security

- **Security Hardening**: Addressed several CodeQL findings related to security and code quality.
- **Type Safety**: Improved type hints and validation logic across critical service paths.
- **Modernized Runtime**: Updated Python compatibility and moved to modern timezone-aware datetime handling.
- **Project Structure**: Refactored core FastAPI application entry points and service registration for better scalability.

## Upgrade and Setup

1. Install updated dependencies: `pip install -r requirements.txt`
2. Run database migrations: `alembic upgrade head`
3. (Optional) Configure new Google or FAL.ai API keys in the Admin panel.
4. Verify your `VERSION` file is updated to `0.3.0-beta`.
5. Start the application: `python -m app.main`

## Known Limitations

- Multi-provider support is stabilized, but specific provider limits and region availability apply.
- ZIP export is currently limited to the active session view.
