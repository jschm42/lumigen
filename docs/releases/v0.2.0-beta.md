# Lumigen v0.2.0-beta

Second release of **Lumigen**, advancing from the initial alpha toward a stable beta with major UI, security, and quality improvements.

> [!NOTE]
> Beta release: core features are stabilized and tests have been expanded, but the public API and data structures may still evolve in minor ways before the first stable release.

## Highlights

- Dynamic gallery with infinite scroll and a floating multi-select panel replaces the static list view.
- Central provider API key management from the admin panel, decoupled from individual model configs.
- Light/dark theme system with dynamic CSS variables and a per-user settings dialog.
- CSRF protection middleware across all state-mutating requests.
- Configurable global upload size limit via `MAX_UPLOAD_SIZE_MB`.
- Comprehensive Playwright end-to-end test suite.
- Full PEP 257 docstring coverage across all public Python classes and methods.

## What's New

### Gallery

- Replaced the bottom-panel gallery layout with a dynamic infinite-scroll feed.
- New floating selection panel for bulk-selecting and acting on multiple assets without leaving the page.
- Extracted gallery filter helper and card fragment to reduce duplication and simplify future changes.
- Added gallery timeframe filters (e.g. last hour, today, this week) for quicker navigation.
- Improved asset detail dialog and confirmation prompt UX.
- Rating styles updated for better visibility in both light and dark themes.

### Admin and Security

- Central API key management: store one API key per provider in the admin panel instead of per model config.
- Fixed API Keys admin tab not rendering due to a missing section registration.
- CSRF token middleware: all POST/PUT/DELETE requests are now CSRF-protected.
- HTMX requests automatically receive the CSRF token via a response header.
- Global upload size limit configurable via `MAX_UPLOAD_SIZE_MB` environment variable.

### Theme and UI

- Full light/dark theme system using dynamic CSS variables and Tailwind class-based dark mode.
- Per-user theme settings dialog accessible from the user menu.
- Refactored CSS into base, light, and dark theme stylesheets for maintainability.
- Improved login page readability in light mode.
- Fixed user menu popup: added `setupUserMenu()` JS toggle and corrected chevron icon states.
- Refactored UI components for improved styling and accessibility.
- Added Bootstrap Icons license and Tailwind font license files.

### Sessions and Workspace

- Extended session time presets for more flexible workflow management.
- Session archiving: long sessions can be archived to keep the workspace tidy.
- Refactored `generate_page` for cleaner workspace view and improved chat data loading.
- Workspace navigation setup added to JavaScript for consistent sidebar behavior.

### Code Quality and Testing

- Added Playwright end-to-end tests covering auth, home, gallery, admin, and profile pages.
- Added PEP 257 docstrings to all public classes and methods across the entire codebase.
- Replaced `datetime.utcnow()` with timezone-aware `datetime.now(UTC)` throughout the codebase.
- Refactored test structure and updated references for UI routes.
- Added linting configuration for Python (`ruff`), JavaScript (`eslint`), HTML (`djlint`), and CSS (`stylelint`).

## Upgrade and Setup

1. Install dependencies: `pip install -r requirements.txt`
2. Run database migrations: `alembic upgrade head`
3. Optionally set `MAX_UPLOAD_SIZE_MB` in `.env` to cap file uploads (default: unlimited)
4. Install Node dependencies for linting or e2e tests: `npm ci`
5. Start the app: `python -m app.main`

## Known Limitations (Beta)

- No stability guarantee for internal APIs and data structures until the first stable release.
- Some provider behavior may vary depending on model and API status.
- Upscaling is optional and depends on local runtime environment and model availability.
