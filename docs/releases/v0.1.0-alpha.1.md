# Lumigen v0.1.0-alpha.1

First public alpha release of **Lumigen**, a local AI image studio web app built with FastAPI + Jinja + HTMX.

> [!WARNING]
> Alpha release: this version is intended for early testing. APIs, data models, and UI details may change in upcoming versions.

## Highlights

- Local end-to-end workflow: enter a prompt, generate an image, manage it in the gallery.
- Multi-provider support for OpenAI, OpenRouter, Google, and BFL.
- Profiles, categories, and session workflows for reproducible generation pipelines.
- Sidecar metadata and request/profile snapshots for better traceability.
- Optional upscaling with Real-ESRGAN.
- Full authentication and role separation for admin/user access.

## Included Features

### Generation and Providers

- Unified provider adapters with registry-driven retry, concurrency, and spacing logic.
- Google image generation (Gemini/Imagen) including model handling and error handling.
- Improved input image support with previews and DataTransfer flow.

### Data, Reproducibility, and Gallery

- Persistent storage for images, thumbnails, and sidecar JSON files.
- Categories replace classic gallery folders, including name-length constraints at DB and app level.
- Optional asset rating.
- Improved gallery filters and UX refinements.

### Admin, Security, and Configuration

- User authentication with role model (`admin`, `user`).
- Model configurations with optional model-specific API keys.
- Encryption for sensitive keys using Fernet (`PROVIDER_CONFIG_KEY`).
- Reverse proxy and HTTPS-related settings for deployment scenarios.

### Quality and Operations

- Alembic migration chain through the user-auth rollout.
- Expanded unit, route, and frontend test coverage.
- Docker improvements including runtime requirements for upscaling.
- Request and provider logging for better diagnostics.

## Upgrade and Setup

1. Install dependencies: `pip install -r requirements.txt`
2. Run database migrations: `alembic upgrade head`
3. Optionally configure session secret and HTTPS/proxy settings in `.env`
4. Start the app: `python -m app.main`

## Known Limitations (Alpha)

- No stability guarantee for internal APIs and data structures.
- Some provider behavior may vary depending on model and API status.
- Upscaling is optional and depends on local runtime environment and model availability.
